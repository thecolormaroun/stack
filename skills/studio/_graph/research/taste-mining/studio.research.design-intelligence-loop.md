# Design Intelligence Loop

This is the rebuilt OpenClaw weekly design digest plus the safer version of taste compounding.

## Purpose

- Pull fresh inspiration from curated design sources.
- Include Maroun's Arc bookmarks every run.
- Include Maroun's Field Theory/X bookmarks every run.
- Check GBrain import deltas so new bookmarks do not disappear between systems.
- Produce review-gated Zettelkasten and Studio/CDO skill candidates.
- Require Codex design eval evidence before a candidate becomes default runtime guidance.

## Source Order

1. Field Theory/X bookmarks - highest personal signal for newly saved design ideas.
2. Arc bookmarks and Arc history - broad saved-link and recent browsing signal.
3. GBrain x-bookmarks and saved-links roots - imported-memory coverage and deltas.
4. Curated web sources - Brian Lovin, Design Spells, Handheld Design, Featured Mobile.

## Weekly Contract

Each run writes a source manifest, a weekly digest, and a promotion packet.

The digest keeps the old three-output shape:
- Output A - Design Digest.
- Output B - Zettelkasten Candidates.
- Output C - Studio Skill Update Candidates.

## Backfill Contract

Initial catch-up window: `2026-04-01` through the current run date, chunked by week.

Reason: the OpenClaw loop has been down for roughly two and a half months, and Field Theory/GBrain deltas show newer X bookmark exports are ahead of imported memory.

Backfill is read-only until a review packet is approved.

## Promotion Contract

Taste compounding starts as proposal-only.

Promote a repeated design rule only after:
- it has evidence from at least two occurrences or a hard eval failure.
- the target file and exact snippet are named.
- an idempotency check says it is not already covered.
- Codex design eval beats `codex-current` on the promotion gate.

Canonical portable skill: `skills/design-intelligence/SKILL.md`.
