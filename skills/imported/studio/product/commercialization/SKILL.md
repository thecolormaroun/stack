# Commercialization

Pricing strategy, GTM planning, and monetization analysis for your products.

## Skill graph entry point
Start at: `./_graph/product/commercialization.moc.md`

## When to Use
- Product is ready to ship and needs pricing
- Evaluating monetization models
- Planning launch strategy

## Frameworks

### Pricing Models
| Model | Best For | Examples |
|-------|----------|---------|
| **Freemium** | User acquisition, network effects | Free tier + paid pro |
| **One-time** | Tools, utilities | Pay once, own forever |
| **Subscription** | Ongoing value, content | Monthly/annual plans |
| **Usage-based** | APIs, infrastructure | Pay per use |

### Pricing Process
1. **Value analysis** — What's you's alternative? What do they pay now?
2. **Cost analysis** — What does it cost to run? (hosting, APIs, time)
3. **Market comp** — What do competitors charge?
4. **Willingness to pay** — Conservative estimate
5. **Margin target** — Minimum viable pricing

### GTM Checklist
- [ ] Landing page with clear value prop
- [ ] Demo/screenshots/video
- [ ] Launch channels identified (PH, HN, Twitter, communities)
- [ ] Pricing page
- [ ] Analytics/tracking
- [ ] Payment integration (Stripe preferred)
- [ ] Email capture for waitlist/updates

### Launch Strategy Template
```markdown

**Target date:** [Date]
**Channels:** [Where to announce]
**Goal:** [Signups/revenue/awareness target]

### Pre-launch
- [ ] Landing page live
- [ ] Beta testers feedback incorporated
- [ ] Launch copy written

### Launch Day
- [ ] Post to [channels]
- [ ] Monitor feedback
- [ ] Respond to comments

### Post-launch (Week 1)
- [ ] Analyze metrics
- [ ] Address top feedback items
- [ ] Plan v1.1
```

## your Preferences
- Start simple (don't over-engineer pricing)
- Conservative revenue estimates
- Prefer Stripe for payments
- Personal projects can be free/open-source — monetization optional
- Work-related tools should have clear ROI story
