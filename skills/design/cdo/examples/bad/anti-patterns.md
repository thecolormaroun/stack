# Design Anti-Patterns to Avoid

These patterns indicate AI-generated or low-quality design work.

## Visual Anti-Patterns

### 1. Purple Gradients Everywhere
❌ `background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);`
Why it's bad: Overused AI default. Screams "generated by Claude/GPT."

### 2. Inter Font as Default
❌ `font-family: 'Inter', sans-serif;`
Why it's bad: Every AI picks Inter. It's safe but signals no design thought.

### 3. Centered Hero with Generic Copy
❌ Large centered headline → subtitle → CTA button
Why it's bad: The most common AI layout. Zero differentiation.

### 4. Three-Column Card Grid
❌ Exactly 3 cards in a row, all same size, all with icon-title-description
Why it's bad: Default AI composition. Real layouts have rhythm and hierarchy.

### 5. Pure Black (#000000) for Text
❌ `color: #000000;`
Why it's bad: Too harsh. Use near-black (#1a1a1a or #0f0f0f) for softer contrast.

### 6. Emoji as Icons
❌ Using 🚀 💡 ⚡ as feature icons
Why it's bad: Lazy. Real products use consistent icon libraries.

### 7. Rounded Corners Everywhere
❌ `border-radius: 16px;` on every element
Why it's bad: Creates visual monotony. Vary radii based on element size and hierarchy.

## Interaction Anti-Patterns

### 8. Bounce Animation on Everything
❌ `animation: bounce 1s infinite;`
Why it's bad: Distracting and amateur. Animation should be purposeful.

### 9. Hover Effects That Move Layout
❌ `&:hover { transform: scale(1.1); }`
Why it's bad: Causes reflow, jarring experience. Use subtle transforms (1.02 max).

### 10. Loading Spinners Instead of Skeletons
❌ Centered spinner for page load
Why it's bad: Modern UX uses skeleton screens that preview layout.

## Code Anti-Patterns

### 11. Magic Numbers
❌ `margin-top: 37px; padding: 23px;`
Why it's bad: No system. Use spacing tokens: `var(--space-lg)`.

### 12. Inline Styles
❌ `style={{ backgroundColor: '#f5f5f5', padding: '20px' }}`
Why it's bad: Unmaintainable. Use CSS classes or styled-components.

### 13. Defensive Comments
❌ `// This is the header component`
Why it's bad: Obvious. Comments should explain *why*, not *what*.

### 14. Any Type Assertions
❌ `const data = response as any;`
Why it's bad: Type safety ignored. Define proper interfaces.

## Structure Anti-Patterns

### 15. No Responsive Consideration
❌ Fixed pixel widths, no breakpoints
Why it's bad: Breaks on mobile. Mobile-first always.

### 16. Missing Focus States
❌ `button:focus { outline: none; }`
Why it's bad: Accessibility violation. Always provide visible focus indicators.

### 17. No Error States
❌ Only happy path designed
Why it's bad: Real UX handles loading, empty, error states.

### 18. Inconsistent Spacing
❌ Mix of 8px, 12px, 15px, 20px, 24px
Why it's bad: No rhythm. Use a spacing scale (4, 8, 12, 16, 24, 32, 48).

## The Fix

Before shipping any UI:
1. Run through this checklist
2. Apply deslop/ skill to remove code smells
3. Apply rams/ skill for accessibility audit
4. Ask: "Would a staff designer approve this?"
