# Design Principles Checklist

Actionable principles from Maroun's vault. Check every design against these.

---

## Visual Design ✓

- [ ] **Does it have taste?** Does this create a pleasing, integrated whole—not just work?
- [ ] **Is it memorable, not just usable?** People fall in love with vibes, not logic
- [ ] **Clear visual hierarchy?** Size, weight, and space create structure
- [ ] **Generous whitespace?** Let content breathe
- [ ] **Every visual choice has a reason?** No gratuitous gradients or glassmorphism
- [ ] **No stock-photo vibes?** Intentional, project-specific assets

---

## Typography ✓

- [ ] **Have I read the content?** Can't design type without understanding the text
- [ ] **Does type honor content?** Servant, not master—invisible infrastructure
- [ ] **~66 characters per line?** Optimal reading rhythm
- [ ] **1.5-1.6 line height for body?** Readable, not cramped
- [ ] **Clear hierarchy through type alone?** Size, weight reveal importance
- [ ] **Using REM not PX?** Respect user accessibility settings
- [ ] **Max 2 font families?** One for headings, one for body (mono is free)
- [ ] **Single word space between sentences?** Not two

---

## UX Flow ✓

### Focus & Clarity
- [ ] **One thing at a time?** Single clear action per step
- [ ] **Primary action obvious?** Secondary actions de-emphasized
- [ ] **No competing CTAs on critical flows?** Remove distraction

### Progress & Momentum
- [ ] **Progress visible?** Users see movement with every action
- [ ] **Celebratory moments at section breaks?** Natural pauses for accomplishment
- [ ] **Easy wins first?** Start with easiest questions, build confidence

### Emotional Barriers
- [ ] **Addressed anxiety points?** Not just functional barriers
- [ ] **Reduced stakes where possible?** "You can change this later"
- [ ] **No jokes about serious topics?** Calm confidence, not playful delight
- [ ] **Acknowledges difficulty?** "Many people find this confusing—here's a simple explanation"

### Tone
- [ ] **Guide, not parent?** Empowering, not condescending
- [ ] **Explanatory, not prescriptive?** "This information helps us..." not "You must provide..."
- [ ] **Shows path forward on errors?** Don't scold, guide correction

---

## Interaction Patterns ✓

- [ ] **Designed for System 1?** Works for overwhelmed, reactive users
- [ ] **Minimal cognitive load?** Fits into existing workflow
- [ ] **Predictable, not surprising?** Smooth, not bouncy
- [ ] **Validates on blur, not keystroke?** Inline validation timing
- [ ] **Empty states designed?** Illustration + CTA, not blank
- [ ] **Loading states shaped like content?** Skeletons, not spinners
- [ ] **Optimistic updates?** Show change immediately
- [ ] **Keyboard shortcuts for power users?** Accessible speed

---

## Systems & Components ✓

- [ ] **Built for reuse?** Not just this use case
- [ ] **Atomic hierarchy clear?** Atoms → Molecules → Organisms
- [ ] **Consistent patterns?** Same problem = same solution
- [ ] **Design system tokens used?** No magic numbers
- [ ] **All states handled?** Default, hover, active, disabled, loading, error, empty
- [ ] **Accessibility built in?** Not an afterthought

---

## Accessibility ✓

- [ ] **Color contrast 4.5:1?** WCAG AA minimum for body text
- [ ] **All interactive elements focusable?** Logical tab order
- [ ] **Focus indicators visible?** Don't remove outlines
- [ ] **Alt text on images?** Icons have labels
- [ ] **Respects prefers-reduced-motion?** Honor user preferences
- [ ] **Touch targets 44px on mobile?** Finger-friendly

---

## AI Interfaces ✓

- [ ] **Vertical, not horizontal?** Specific workflow, not "do anything"
- [ ] **Guided, not open-ended?** Structure over blank prompt box
- [ ] **Right abstraction level?** Not too complex, not hiding useful control
- [ ] **Mental models established?** Users understand effect on output, not technical details
- [ ] **Human in the loop for high stakes?** Don't trust magic answers blindly
- [ ] **Dynamic UI that adapts?** Shows relevant controls for current task

---

## Before Shipping ✓

- [ ] **Friction logged the flow?** Used it as a new user would
- [ ] **Every micro-frustration addressed?** They compound
- [ ] **Know your customers deeply?** Use product more than they do
- [ ] **No placeholder content in production?** No lorem ipsum that ships
- [ ] **Dark mode checked?** Reduce font weight, use off-white not pure white
- [ ] **Mobile reviewed?** Touch targets, stack layouts, simplified

---

## Design Leadership Check ✓

- [ ] **Can we do this with quality?** Say no to under-resourced work
- [ ] **Thinking horizontally?** Does this affect other teams/flows?
- [ ] **Prototyped the full experience?** Not just the product, but marketing, unboxing, support
- [ ] **Craft, not process?** Quality comes from judgment, not methodology adherence
